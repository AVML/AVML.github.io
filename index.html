About AVML

Audio Visual Markup Language (AVML) is the standard markup language for creating media pages and media optimised applications. Media browsers receive AVML documents from a web server or from local storage and render the documents into rich user interfaces. AVML describes the structure of a media page semantically. The appearance is decided by the browser.

AVML documents MUST be encoded using JSON-ST. AVML media types are the building blocks of AVML documents/pages. They provide a means to create structured media documents by denoting semantics such as title, description, parts(i.e. episodes, series), cover images and other items.

At the root of any media representation is the Media type which may be wrapped by super media types such Series -> SeriesPart or Live. Media type itself may contain other Media type elements (i.e. related media such trailers) Lower level media types alone such Stream or Part without being wrapped by a Media type may not be supported by browsers.

To describe a list of unrelated media types in a single page an array should may be used as below 

[
 {"@ref": "https://example.com/media1.jsonst"}
 {"@ref": "https://example.com/media2.jsonst"}
 {"@ref": "https://example.com/media3.jsonst"}
]

Series

Series represents a tv series. 


type Series struct {
	// The series parts(i.e. seasons)
	Parts []SeriesPart
}

SeriesPart

SeriesPart represents a series part/season. 


// SeriesPart Represents a series part/season
type SeriesPart struct {
	// e.g. Episodes
	Parts []Media
}

Live

Live represents a live media stream. 


type Live struct {
	Media
	// The media here is actually the program
	// name (e.g. ABC-News) along with the current
	// media stream playing. IN the future we may
	// want to replace Media with a custom type.
	// This is basically a scheduled playlist
	Guide []Guide
}

Media

Media represents a media container. 


type Media struct {
   // Breadcrumb trail provides hierarchical
   // structure of the media path so that if presented 
   // as stand-alone it can be identified as part 
   // of a super media type such Series
   // https://en.wikipedia.org/wiki/Breadcrumb_(navigation)
   Breadcrumbs []Header

   // Media Content 
   Part MediaPart 
   Meta
}

Header

Header represents media part header. 


type Header struct {
	ContentID   string
	Name        string
	Number      int
	Description string
	Air         time.Time
	// Trailers/Related media content
	Related []MediaPart 
}

Stream

Stream represents a audio/video stream. 


type Stream struct {
	Index int
	// Iso639_1
	Language string
	Label    string
	Codec    Codec
   enum{
      VideoSpec
      AudioSpec
   }
	Disposition Disposition
	Bitrate     string
}

Guide

Guide represents the tv guide of a live program. 


type Guide struct {
	Start time.Time
	Stop  time.Time
	Media
}

type MediaPart struct {

}

Part

Part represents a media part(i.e. an audio/video file). 


type Part struct {
	mime.Mime
	Streams []Stream
	Format  Format
}

Meta

Meta represents met info of the stream. 


type Meta struct {
	Genre   []string

}

Codec 


type Codec struct {
   Type CodecType

VideoSpec video specs of the strem. 


type VideoSpec struct {
	// 24000/1001
	FrameRate string
	// 1912
	Width int 
	// 792
	Height int 
	//  1:1
	SampleAspectRatio string

AudioSpec

AudioSpec audio specs of the strem. 


type AudioSpec struct {
	// 48000
	SampleRate int64 `json:",omitempty"`
	// 6
	Channels int `json:",omitempty"`
	// 5.1(side)
	ChannelLayout string `json:",omitempty"`
}
