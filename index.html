About AVML

Audio Visual Markup Language (AVML) is the standard markup language for creating media pages and media optimised applications. Usually media browsers receive AVML documents from a web server or from local storage and render the documents into rich user interfaces. AVML describes the structure of a media page semantically. The appearance is decided by the browser.

At the root of any media representation is the Media type which may be wrapped by super media types such Series, Live etc. Media type itself may contain other Media type elements (i.e. related media such trailers) Lower level media types alone such Stream or Part without being wrapped by a Media type may not be supported by browsers.

AVML documents MUST be encoded using JSON-ST. AVML media types are the building blocks of AVML documents/pages. They provide a means to create structured media documents by denoting semantics such as title, description, parts(i.e. episodes, series), cover images and other items.

AVML media type is "application/av-ml".

Within JSON-ST documents the types must be referenced using a fully qualified namespace. The template of the namespace for AV-ML documents is av-ml.org/#{$Type} where {$Type}

All date fields must use rfc3339 format

Types list

Series

Series represents a tv series. 

   type Series struct {
      Name        string
      Description string
      Image       []Image
      Related []Related
      Parts []SeriesPart
      Meta MetaSeries
   }

SeriesPart

SeriesPart represents a series part/season. 


   // SeriesPart represents a series part/season
   type SeriesPart struct {
      // Season
      Header
      // Episodes
      // Type Media
      Parts List
      Meta MetaSeries

   }

	Genre   []string

Part

Part represents a media part(i.e. a audio/video file). 


 
   type Part struct {
      Content-ID string
      Content-Type ContentType
      Streams []Stream
      Format  Format
      Ad Ad
      License License
   }

type ContentType struct {
	Type       string
	Subtype    string
	Parameters []NameValue
}

// e.g. Charset -> UTF8
type NameValue struct {
	Name  string
	Value string
}

type Ad struct{
   // Vast URI tag
   Tag string
}

type License struct{
   // License uri
   Content-ID string
}

type Kind enum {

Meta

Meta represents met info of the stream. 


type Meta struct {
	Genre   []string
	Kind    Kind
	Country []string 
	Credits []Credit
   Release date
   Rating  []Rating    `json:",omitempty"`
	Review  Review    `json:",omitempty"`
}

// Rating system aimed at giving parents
// the information they need to decide
// whether a film is appropriate for their family.
type Rating struct {
	// codes provided by different organisations such MPA
	// filmratings.com
	// e.g.  PG-13, NC-17
	Code string
}


type Review struct {
	// Overall rating score
	// based on the ratings received
	Score    float
	Messages []ReviewMessage
}

type ReviewMessage struct {
	// Review title
	Name string
	// Review body
	Body string
	// A score from 1 to 10
	// where 10 is the highest score
   Score uint
   Date date
}

Image

Image 

              

type Image struct {
	Content-ID string
	Width     uint
	Height    uint
	Role      enum{
	   // poster replicates the movie posters often seen in cinema complexes,
	   // or the front cover of home video releases, and contain a clearly
	   // visible logo or name of the video.
	   "poster"
	   // poster-clear is a poster without text or a logo.
	   "poster-clear"
	   // landscape is a Poster with a wide aspect ratio
	   // e.g. 16:9
	   "landscape"
	   // landscape-clear is a landscape without text or a logo.
	   "landscape-clear"
	   // Used as full screen displays.
	   "fullscreen"
	   // fullscreen-clear is a fullscreen without text or a logo.
	   "fullscreen-clear"
   }
}
