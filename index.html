About AVML

Audio Visual Markup Language (AVML) is the standard markup language for creating media pages and media optimised applications. Usually media browsers receive AVML documents from a web server or from local storage and render the documents into rich user interfaces. AVML describes the structure of a media page semantically. The appearance is decided by the browser.

AVML documents MUST be encoded using JSON-ST. AVML media types are the building blocks of AVML documents/pages. They provide a means to create structured media documents by denoting semantics such as title, description, parts(i.e. episodes, series), cover images and other items.

At the root of any media representation is the Media type which may be wrapped by super media types such Series, Live etc. Media type itself may contain other Media type elements (i.e. related media such trailers) Lower level media types alone such Stream or Part without being wrapped by a Media type may not be supported by browsers.

Within JSON-ST documents the types must be referenced using a fully qualified namespace. The template of the namespace is av-ml.org/#{$Type} where {$Type} is the existing avml type.

Examples

Types list

Series

Series represents a tv series. 

   type Series struct {
      Name        string
      Description string
      Image       []Image
      // URI list of type Related
      Related []string
      // URI list of type SeriesPart
      Parts []string
   }

SeriesPart

SeriesPart represents a series part/season. 


   // SeriesPart represents a series part/season
   type SeriesPart struct {
      // e.g. Episodes
      Header
      // Type Media
      Parts []string
   }

Live

Live represents a live media stream. 

   type Live struct {
      Part MediaPart
      // The media here is actually the program
      // name (e.g. FOXNews) along with the current
      // media stream playing. In the future we may
      // want to replace Media with a custom type.
      // This is basically a scheduled playlist
      Guide []Guide
   }

Media

Media represents a media container. 

type MediaPart struct {
	Part
	// Containers that should to be muxed with the streams within Part
	// on play. E.g. external subtitles
	External []Part
}

Header

Header represents media part header. 


   type Header struct {
      Name        string
      Number      int
      Description string 
      Air         time
      // URI of type Related to Trailers/Related media content
      Related []string
      Image   []Image
   }

Stream

Stream represents a audio/video stream. 


type Stream struct {
	// Iso639_1
	Language string
	Label    string
	Codec    Codec
   enum{
      VideoSpec
      AudioSpec
   }
	Disposition Disposition
	Bitrate     string
}

Guide

Guide represents the tv guide of a live program. 

   type Guide struct {
      // When the program
      // is scheduled to start
      Start time.Time

      Header

      Meta
   }

MediaPart

 


   type MediaPart struct {
      Part
      // Containers that should to be muxed with the streams within Part
      // on play. E.g. an external subtitles
      External []Part
   }

Part

Part represents a media part(i.e. a audio/video file). 


 
   type Part struct {
      Content-ID string
      Content-Type ContentType
      Streams []Stream
      Format  Format
   }

type ContentType struct {
	Type       string
	Subtype    string
	Parameters []NameValue
}

// e.g. Charset -> UTF8
type NameValue struct {
	Name  string
	Value string
}

Kind

Kind represents the kind of the stream. 


type Kind enum {
	Video
	Music
	MusicVideo
	Radio
}

Meta

Meta represents met info of the stream. 


type Meta struct {
	Genre   []string
	Kind    Kind
	Country []string
	Rating  string
	Credits []Credit
	Release time.Time
}

Codec

Codec 


type Codec struct {
   Type CodecType

   // h264 for video or ac3 for audio
   Name string

   // H.264 / AVC / MPEG-4 AVC / MPEG-4 part 10
   // or  ATSC A/52A (AC-3)
   LongName string
}

VideoSpec

VideoSpec video specs of the strem. 


type VideoSpec struct {
	// 24000/1001
	FrameRate string
	// 1912
	Width int 
	// 792
	Height int 
	//  1:1
	SampleAspectRatio string 
	// 239:99
	DisplayAspectRatio string 
}

AudioSpec audio specs of the strem. 


type AudioSpec struct {
	// 48000
	SampleRate int64
	// 6
	Channels int
	// 5.1(side)
	ChannelLayout string 
}

Collection

Collection 

   type Collection struct {
      Title       string
      Description string
      Image       []Image
      Items  []Item
      // optional
      Hits uint64
   }   

   type Item struct {
      @id         string
      @type enum{
         // a qualified type name using the template
         // av-ml.org/#{$Type} where $type is an existing avml type
         // e.g. av-ml.org/tag.Media, av-ml.org/tag.Collection etc
         av-ml.org/#{$Type}
      }
   }

type Region struct {

}
